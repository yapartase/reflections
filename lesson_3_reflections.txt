>>When would you want to use a remote repository rather than keeping all your
work local?

when working with other people on the same repo.

>>Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

You might not want to share all changes yet with the community.

>>Describe the differences between forks, clones, and branches. When would you
use one instead of another?

Forking is the github term for cloning. Forks exist online in the github world
whereas cloning occurs locally. Branches refer to a offspring of one of the
former nodes. They do not affect the old path unless they are merged.

>>What is the benefit of having a copy of the last known state of the remote
stored locally?

You can compare with your own current branch and check for changes etc.
